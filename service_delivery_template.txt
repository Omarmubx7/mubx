2. SECURITY CHECKLIST
------------------------------------------------------------------------------
[ ] ENVIRONMENT VARIABLES
    - [ ] .env.local created and added to .gitignore
    - [ ] API Keys (Supabase, Stripe, etc.) are never hardcoded.

[ ] HEADERS & CONFIG
    - [ ] next.config.ts configures security headers (X-Content-Type-Options, X-Frame-Options).
    - [ ] Content Security Policy (CSP) configured if handling sensitive data.

[ ] VALIDATION
    - [ ] Zod schemas implemented for all form inputs.
    - [ ] API routes validate request body and method.
    - [ ] Rate limiting enabled for public APIs (using Upstash or similar).

[ ] DATABASE (If Applicable)
    - [ ] Row Level Security (RLS) policies enabled.
    - [ ] No public write access.

------------------------------------------------------------------------------
3. SEO & DISCOVERABILITY
------------------------------------------------------------------------------
[ ] METADATA (src/config/seo.ts or page.tsx)
    - [ ] Title Tag < 60 characters (e.g., "Brand Name | Keyword - City").
    - [ ] Meta Description < 120 characters, actionable and clear.
    - [ ] Open Graph Images (1200x630px) generated and linked.
    - [ ] Canonical URL set to prevent duplicate content issues.

[ ] TECHNICAL SEO
    - [ ] sitemap.xml generated (using next-sitemap or custom route).
    - [ ] robots.txt configured to allow indexing (User-agent: * Allow: /).
    - [ ] Semantic HTML used (h1, h2, nav, main, footer).
    - [ ] Images have descriptive 'alt' text.

[ ] SCHEMA MARKUP (JSON-LD)
    - [ ] Organization/LocalBusiness schema added.
    - [ ] 'Service' or 'Product' schema added for main offerings.
