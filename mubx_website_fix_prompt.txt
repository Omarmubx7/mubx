### Issue #8: Button Nested in Link (Invalid HTML)
**Problem**: 
- `<a>` tags contain `<button>` elements
- This is invalid HTML5 - interactive elements shouldn't nest

**Location**: Hero CTA buttons

**Impact**: Unpredictable behavior, keyboard navigation issues, fails HTML validation

**Fix Required**:
```html
<!-- Current (INVALID) -->
<a href="https://calendly.com/...">
  <button>GET IN TOUCH</button>
</a>

<!-- Option 1: Use button with onclick -->
<button onclick="window.location.href='https://calendly.com/...'">
  GET IN TOUCH
</button>

<!-- Option 2: Style link as button (RECOMMENDED) -->
<a href="https://calendly.com/..." class="button-link" role="button">
  GET IN TOUCH
</a>
```

**Acceptance Criteria**:
- [ ] No buttons nested inside links
- [ ] Maintain visual button styling
- [ ] Keyboard navigation works correctly (Tab, Enter)
- [ ] Passes HTML5 validation

## 📱 PHASE 6: MOBILE/PERFORMANCE OPTIMIZATIONS

### Issue #18: Top Marquee Mobile Performance
**Problem**: 
- Infinite scrolling animation may cause performance issues on mobile
- Battery drain and choppy animation on slower devices

**Location**: Top services ticker

**Impact**: Poor mobile UX, high battery consumption, jank

**Fix Required**:
```css
/* Enable hardware acceleration */
.marquee {
  will-change: transform; /* Hint to browser */
  transform: translateZ(0); /* Force GPU */
  animation: scroll 30s linear infinite;
}

/* Pause on devices with reduced performance */
@media (prefers-reduced-motion: reduce) {
  .marquee {
    animation-play-state: paused;
  }
}

/* Pause when not visible */
.marquee:not(.in-viewport) {
  animation-play-state: paused;
}
```

**JavaScript Optimization**:
```javascript
// Pause animation when off-screen using Intersection Observer
const observer = new IntersectionObserver((entries) => {
  entries.forEach(entry => {
    if (entry.isIntersecting) {
      entry.target.style.animationPlayState = 'running';
    } else {
      entry.target.style.animationPlayState = 'paused';
    }
  });
});

observer.observe(marqueeElement);
```

**Acceptance Criteria**:
- [ ] 60fps animation on mid-range mobile devices
- [ ] Animation pauses when off-screen
- [ ] No layout thrashing
- [ ] Test on actual mobile devices (not just Chrome DevTools)
