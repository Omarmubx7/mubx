"Act as a Senior Web Architect. I need a corporate/business website built with Next.js 15 and a Headless CMS.
The goal is to build trust and authority. The client must be able to edit blog posts and team members easily.

# Project Specs
- **Framework**: Next.js 15 (App Router)
- **CMS**: Sanity.io (preferred) or Contentful.
- **Styling**: Tailwind CSS
- **SEO**: `next-sitemap` + structured data (LD-JSON).
- **Localization**: Prepared for i18n (en/ar) routing structure.

# Content Architecture (CMS Schemas)
1. **Post**: Title, Slug, Main Image, Body (Portable Text), Categories, Author.
2. **Author**: Name, Image, Bio.
3. **Service**: Title, Description, Icon Name, Price (optional).
4. **TeamMember**: Name, Role, Image, Linkedin URL.

# Site Map & Layout
- **Navbar**: Logo, Links (Home, About, Services, Blog, Contact), CTA ('Get Quote').
- **Home**: Hero, Service Highlights, Latest Insights (Blog), Partners.
- **Services Index**: Grid of all services.
- **Service Detail** (dynamic `[slug]`): Detailed explanation + FAQ + CTA.
- **About**: Company Story, Values, Team Grid.
- **Contact**: Map, Info, Form.

# Technical Requirements
1. **ISR**: Use Incremental Static Regeneration (`revalidate: 60`) for CMS pages.
2. **Images**: Use `sanity-image` helper for hot-spot cropping and automatic optimization.
3. **SEO**: Auto-generate sitemap.xml from CMS slugs.

Please start by defining the Sanity schemas and the Next.js folder structure for these routes."
